# КМУ № 373 (2021) - Вимоги до кібербезпеки об'єктів критичної інфраструктури

**Status:** Діючий
**Тип:** Постанова Кабінету Міністрів України
**Дата набрання чинності:** 2021-05-19
**Повна назва:** "Про затвердження Загальних вимог до кіберзахисту об'єктів критичної інфраструктури"

## Сфера застосування

Встановлює загальні вимоги до кіберзахисту для:

- Об'єктів критичної інфраструктури (згідно ЗУ № 1882-IX)
- Автоматизованих систем управління технологічними процесами (АСУ ТП)
- Промислових систем керування (ICS/SCADA)

## Ключові вимоги

### 1. Організаційні заходи

- Призначення відповідальних за кіберзахист
- Розробка політики кібербезпеки
- Реєстрація об'єкта в Держспецзв'язку (якщо потрібно)

### 2. Технічні заходи

- Сегментація мережі (OT/IT розділення)
- Засоби захисту периметру (firewall, IDS/IPS)
- Контроль доступу та автентифікація (обов'язкова MFA для admin)
- Моніторинг подій безпеки
- Резервне копіювання

### 3. Управління інцидентами

- План реагування на кіберінциденти (IRP - Incident Response Plan)
- Процедури повідомлення про інциденти
- Відновлення після інцидентів (BCP/DRP)

### 4. Захист каналів зв'язку

- Шифрування трафіку (TLS 1.3+, IPsec VPN)
- Захист протоколів (MQTT, Modbus, OPC-UA)
- Сертифікати та PKI

### 5. Оновлення та patch management

- Регулярні оновлення ПЗ
- Керування вразливостями
- Політика застарілого ПЗ

## Ключові терміни

- **Об'єкт критичної інфраструктури** - об'єкт, порушення функціонування якого може завдати шкоди національній безпеці
- **Кіберінцидент** - подія в інформаційній системі, що може призвести до порушення безпеки
- **Сегментація мережі** - розділення мережі на ізольовані сегменти для обмеження поширення загроз
- **Двофакторна автентифікація (MFA)** - обов'язкова для адміністраторів та диспетчерів

## KRTM Usage

**Том 2 (Основні рішення):**

- Розділ "Кібербезпека" - архітектура згідно КМУ 373

**Security Architecture (Section 6 TZ):**

1. **Сегментація мережі:**
   - OT Network (котельні, Edge Nodes, Modbus/MQTT)
   - IT Network (диспетчерські робочі станції, бази даних)
   - DMZ (VPN concentrator, зовнішні інтеграції)

2. **VPN-тунелі для Edge Nodes:**
   - Шифрований зв'язок 65 котелень → ЦДП
   - MQTT over TLS 1.3+ всередині VPN
   - Окремий тунель для кожного Edge Node

3. **Контроль доступу:**
   - RBAC в ThingsBoard (ролі: адмін, диспетчер, оператор, read-only)
   - **Обов'язкова MFA** для admin та dispatcher accounts (TOTP)
   - Унікальні токени/X.509 сертифікати для кожного Edge Node

4. **Моніторинг безпеки:**
   - Centralized logging (syslog → 1+ year retention)
   - SIEM або Security dashboard в Grafana
   - Alerting на anomalies (Rule Engine в ThingsBoard)

5. **Incident Response:**
   - IRP document (зразок у Section 6.4 TZ)
   - BCP/DRP з RPO/RTO targets
   - Emergency contacts

6. **Hardening:**
   - CIS Benchmarks для Ubuntu Server
   - Minimal service exposure (принцип least privilege)
   - Automated security updates (unattended-upgrades)

## Compliance Notes

**Critical for KRTM Project:**

- ✅ KRTM може підпадати під критичну інфраструктуру (теплопостачання міста 650,000 населення)
- ✅ Координація з Держспецзв'язком (реєстрація об'єкта - можлива вимога)
- ✅ Обов'язковий IRP, BCP/DRP для критичної інфраструктури

**Risk Assessment:**

- Якщо KRTM визнано критичною інфраструктурою → повний compliance з КМУ 373
- Якщо НЕ критична → рекомендується слідувати best practices (IEC 62443 Level 2)

## Relation to IEC 62443

КМУ 373 гармонізований з **IEC 62443** (Industrial Cybersecurity):

- Security Levels: КМУ 373 ≈ IEC 62443 SL2 (захист від цілеспрямованих атак з обмеженими ресурсами)
- Defense in Depth: layers (network, host, application, data)
- Зонування: Conduit і Zone model з IEC 62443
