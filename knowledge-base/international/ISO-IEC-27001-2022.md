# ISO/IEC 27001:2022 - Information Security Management Systems

**Status:** International Standard (Reference for KRTM)
**Full name:** ISO/IEC 27001:2022 - Information security, cybersecurity and privacy protection — Information security management systems — Requirements
**Latest edition:** 2022-10 (replaces ISO 27001:2013)
**Application in Ukraine:** Voluntary certification (widely recognized)

## Сфера застосування

Встановлює вимоги до системи управління інформаційною безпекою (ISMS - Information Security Management System):

- Захист конфіденційності, цілісності, доступності інформації
- Управління ризиками інформаційної безпеки
- Continuous improvement process (PDCA cycle)

## Структура стандарту

**Main Clauses:**
1-3. Scope, References, Terms 4. Context of the organization 5. Leadership (management commitment) 6. Planning (risk assessment, risk treatment) 7. Support (resources, competence, awareness) 8. Operation (implement controls) 9. Performance evaluation (monitoring, audit) 10. Improvement (nonconformity, corrective actions)

**Annex A: Controls (93 controls in ISO 27001:2022):**

- Organizational (37 controls)
- People (8 controls)
- Physical (14 controls)
- Technological (34 controls)

## Key Concepts

**CIA Triad:**

- **Confidentiality** - інформація доступна тільки авторизованим особам
- **Integrity** - інформація точна та повна
- **Availability** - інформація доступна коли потрібно

**Risk Management Process:**

1. **Risk Identification** - що може піти не так?
2. **Risk Assessment** - яка ймовірність і impact?
3. **Risk Treatment** - accept, mitigate, transfer, avoid
4. **Risk Monitoring** - continuous review

**PDCA Cycle:**

- **Plan:** Establish ISMS, assess risks
- **Do:** Implement controls
- **Check:** Monitor and review
- **Act:** Improve and correct

## Annex A Controls (Selected for KRTM)

### Organizational Controls (приклади):

- **A.5.1:** Policies for information security
- **A.5.7:** Threat intelligence
- **A.5.23:** Information security for use of cloud services
- **A.5.30:** ICT readiness for business continuity (BCP/DRP)

### People Controls:

- **A.6.1:** Screening (background checks)
- **A.6.2:** Terms and conditions of employment
- **A.6.3:** Information security awareness, education and training

### Physical Controls:

- **A.7.2:** Physical entry controls (серверна ЦДП, котельні)
- **A.7.4:** Physical security monitoring (video surveillance)
- **A.7.7:** Clear desk and clear screen
- **A.7.14:** Secure disposal or re-use of equipment

### Technological Controls (критичні для KRTM):

- **A.8.2:** Privileged access rights (admin accounts)
- **A.8.3:** Information access restriction (RBAC)
- **A.8.5:** Secure authentication (MFA mandatory)
- **A.8.9:** Configuration management (CaC - Configuration as Code)
- **A.8.15:** Logging (centralized, 1+ year retention)
- **A.8.16:** Monitoring activities
- **A.8.19:** Installation of software on operational systems (patch mgmt)
- **A.8.20:** Networks security (segmentation, firewall)
- **A.8.22:** Segregation in networks (OT/IT separation)
- **A.8.23:** Web filtering
- **A.8.24:** Use of cryptography (TLS 1.3+)

## KRTM Usage

**ISO 27001 для KRTM - voluntary, але highly recommended:**

**Section 6 (Cybersecurity) TZ references ISO 27001 controls:**

1. **Access Control (A.8.x):**
   - RBAC в ThingsBoard (admin, dispatcher, operator, read-only)
   - MFA для admin та dispatcher accounts (TOTP)
   - Унікальні X.509 сертифікати для кожного Edge Node

2. **Cryptography (A.8.24):**
   - TLS 1.3+ для MQTT (Edge → ЦДП)
   - VPN-тунелі (IPsec або WireGuard)
   - Certificate management (PKI або self-signed для internal)

3. **Logging and Monitoring (A.8.15, A.8.16):**
   - Centralized logging (syslog → 1+ year retention)
   - NTP/PTP time synchronization (для forensics)
   - Security monitoring через Grafana або SIEM

4. **Network Security (A.8.20, A.8.22):**
   - Segmentation: OT (котельні) / IT (диспетчерські) / DMZ
   - Firewall rules між зонами
   - VPN enforcement для external access

5. **Configuration Management (A.8.9):**
   - Configuration as Code (Git repository)
   - CI/CD pipeline з secret scanning та SBOM
   - Peer review та merge requests (CODEOWNERS)

6. **Business Continuity (A.5.30):**
   - BCP/DRP з RPO/RTO targets
   - HA configuration для ThingsBoard ЦДП
   - 7-day buffer на Edge Nodes (failover при втраті зв'язку)

7. **Physical Security (A.7.x):**
   - Контроль доступу до серверної ЦДП
   - Замки на шафах ТМ у котельнях
   - Frigate NVR (відеоспостереження) - DPIA compliance required

8. **Incident Response (A.5.24-5.28):**
   - IRP (Incident Response Plan) - обов'язковий
   - Процедури реагування на кіберінциденти
   - Координація з КМУ 373 requirements

## Certification Context

**ISO 27001 Certification:**

- Audit by accredited certification body (TÜV, BSI, SGS тощо)
- 3-year certificate with annual surveillance audits
- Expensive (€10k-50k залежно від scope)

**For KRTM:**

- ✅ Full certification - NOT required (занадто дорого для муніципального проекту)
- ✅ Self-assessment against Annex A controls - RECOMMENDED
- ✅ Use ISO 27001 як framework для Section 6 (Cybersecurity) documentation
