## 7.1. Загальні положення

### 7.1.1. Поняття ПНР

**Пусконалагоджувальні роботи (ПНР)** - комплекс робіт з перевірки, налаштування та випробування технологічного обладнання, систем автоматизації та диспетчеризації для забезпечення їх роботоздатності згідно проектних параметрів.

**Мета ПНР:**

- Перевірка працездатності обладнання після монтажу
- Налаштування параметрів роботи
- Інтеграція окремих компонентів в єдину систему
- Випробування системи в цілому
- Навчання експлуатаційного персоналу

### 7.1.2. Відмінності ПНР від монтажу

| Аспект           | Монтажні роботи              | ПНР                               |
| ---------------- | ---------------------------- | --------------------------------- |
| **Мета**         | Фізична установка обладнання | Налаштування та введення в роботу |
| **Результат**    | Змонтоване обладнання        | Працююча система                  |
| **Од. виміру**   | Фізичні (шт, м, компл)       | Функціональні (система, об'єкт)   |
| **Кваліфікація** | Монтажники                   | Інженери-налагоджувачі            |
| **ЗВВ**          | 75-95%                       | 120-140%                          |
| **КП**           | 50-65%                       | 65-75%                            |

### 7.1.3. Класифікація ПНР

**За видами систем:**

1. **Електротехнічні системи:**
   - Електропостачання (підстанції, розподільчі щити)
   - Освітлення
   - Заземлення та блискавкозахист

2. **Системи автоматизації та диспетчеризації:**
   - SCADA/HMI системи
   - PLC (програмовані логічні контролери)
   - IoT платформи (ThingsBoard, Node-RED тощо)
   - Промислові протоколи (Modbus, OPC-UA, MQTT)

3. **Системи теплопостачання:**
   - Котельні установки
   - Теплові пункти
   - Системи обліку тепла

4. **Мережеві та телекомунікаційні системи:**
   - Локальні мережі (LAN)
   - VPN з'єднання
   - Системи відеоспостереження (NVR)

**KRTM:** Комбінація типів 2, 3, 4 (IoT диспетчеризація теплопостачання з мережевою інфраструктурою)

## 7.2. Склад пусконалагоджувальних робіт

### 7.2.1. Етапи ПНР

**Етап 1: Підготовчі роботи**

- Вивчення проектної та технічної документації
- Перевірка комплектності обладнання
- Перевірка якості монтажу
- Підготовка інструментів та вимірювальної апаратури

**Етап 2: Автономні випробування**

- Перевірка окремих компонентів системи
- Налаштування параметрів
- Вимірювання характеристик
- Усунення виявлених дефектів

**Етап 3: Комплексні випробування**

- Перевірка взаємодії всіх компонентів
- Імітація робочих режимів
- Тестування аварійних ситуацій
- Перевірка захистів та блокувань

**Етап 4: Приймальні випробування**

- Робота системи в реальних умовах (тестовий режим)
- Вимірювання проектних параметрів
- Оформлення документації
- Навчання персоналу

**Етап 5: Дослідна експлуатація**

- Робота під наглядом налагоджувачів (30+ днів)
- Усунення виявлених недоліків
- Оптимізація параметрів
- Передача в промислову експлуатацію

### 7.2.2. KRTM: Специфіка ПНР для IoT системи

**Edge Node (типова котельня):**

**Етап 1: Підготовка (0.5 дні)**

- Перевірка монтажу обладнання в шафі ТМ
- Перевірка прокладки кабелів RS-485 до КВП
- Перевірка живлення 220V, тест UPS

**Етап 2: Налаштування Gateway (1 день)**

- Встановлення ОС (Raspberry Pi OS Lite)
- Встановлення ThingsBoard Gateway (Docker)
- Конфігурація Modbus Connector:
  ```yaml
  # gateway.yaml (приклад)
  connectors:
    - name: 'Modbus Connector'
      type: 'modbus'
      configuration: 'modbus.json'
  ```
- Конфігурація підключення КВП (slave ID, регістри, polling interval)

**Етап 3: Інтеграція з ЦДП (1 день)**

- Генерація сертифікатів X.509 для MQTT over TLS
- Налаштування MQTT broker ЦДП (адреса, порт, credentials)
- Тестування з'єднання (ping, telnet, openssl s_client)
- Перевірка передачі телеметрії (температура, тиск, витрата)

**Етап 4: Комплексні тести (1 день)**

- Тест зчитування даних з 10 КВП (Modbus RTU)
- Тест передачі до ЦДП (MQTT, QoS 1)
- Тест локального буферування (відключення ЦДП на 1 год)
- Тест відмовостійкості:
  - Відключення живлення 220V → UPS
  - Відключення 4G мережі → автоматичне відновлення
  - Перезавантаження Raspberry Pi → автозапуск Gateway

**Етап 5: Навчання персоналу (0.5 дні)**

- Начальник котельні: базовий огляд системи
- Інструкції з перезапуску обладнання
- Контактні дані технічної підтримки

**Разом на 1 котельню:** 4 дні (інженер-налагоджувач)

**ЦДП (Центральний Диспетчерський Пункт):**

**Етап 1: Налаштування ThingsBoard кластера (5 днів)**

- Встановлення PostgreSQL (master-slave реплікація)
- Встановлення Cassandra (якщо використовується для time-series)
- Встановлення ThingsBoard CE (HA режим, 2+ ноди)
- Налаштування Load Balancer (nginx або HAProxy)

**Етап 2: Налаштування Frigate NVR (2 дні)**

- Встановлення Frigate (Docker)
- Інтеграція з IP камерами (RTSP streams)
- Налаштування AI детекції (YOLO моделі)
- Інтеграція з ThingsBoard (MQTT alarms)

**Етап 3: Створення Dashboard (3 дні)**

- Карта всіх 65 котелень (Leaflet, OpenStreetMap)
- Віджети: температура, тиск, витрата (real-time)
- Графіки: часові ряди (time-series), KPI/SEC
- Алярми: межі по параметрах, події з камер

**Етап 4: Rule Engine (2 дні)**

- Правила валідації даних (range checks, outliers)
- Правила алярмів (перевищення меж, втрата зв'язку)
- Правила розрахунків (KPI, SEC, агрегація)

**Етап 5: Комплексні тести ЦДП (3 дні)**

- Тест навантаження (65 Edge Nodes × 10 датчиків × 1 хв інтервал = 650 msg/min)
- Тест відмовостійкості (вимкнення однієї ноди ThingsBoard → автоматичний failover)
- Тест відновлення (crash PostgreSQL → відновлення з резервної копії)

**Етап 6: Навчання диспетчерів (5 днів)**

- Робота з Dashboard (перегляд даних, графіків)
- Обробка алярмів
- Перегляд відеоархіву (Frigate)
- Базове адміністрування (створення користувачів, налаштування доступу)

**Разом ЦДП:** 20 днів (команда: 2 інженери ThingsBoard + 1 інженер Frigate)

**Дослідна експлуатація (Pilot Operation):**

- Мінімум 30 днів (згідно Tom 1, Section 8)
- Всі 65 котелень + ЦДП працюють під наглядом
- Логування помилок, оптимізація параметрів
- Критерій успіху: 99% uptime, відсутність критичних помилок

## 7.3. Визначення вартості ПНР

### 7.3.1. Методи визначення вартості

**Метод 1: Ресурсний (рекомендований)**

**Формула:**

```
ПВ_ПНР = ФОП_інженерів + ЄСВ (22%) + Витратні матеріали + Амортизація інструментів
```

де:

- **ФОП_інженерів** = Кількість інженеро-днів × Денна ставка
- **ЄСВ** = 22% від ФОП
- **Витратні матеріали** - кабелі для тестування, з'єднувачі, запчастини
- **Амортизація** - ноутбуки, осцилографи, мультиметри, генератори сигналів

**KRTM Приклад (1 котельня):**

```
Інженер-налагоджувач (категорія II): 4 дні × 3,000 грн/день = 12,000 грн
ЄСВ (22%): 2,640 грн
Витратні матеріали (патч-корди, адаптери): 500 грн
Амортизація інструментів (ноутбук, тестер): 200 грн

ПВ_ПНР = 12,000 + 2,640 + 500 + 200 = 15,340 грн

ЗВВ (125% від ФОП): 12,000 × 1.25 = 15,000 грн
КП (70% від ФОП): 12,000 × 0.70 = 8,400 грн

Вартість ПНР (1 котельня) = 15,340 + 15,000 + 8,400 = 38,740 грн
```

**Метод 2: Нормативний (за довідниками)**

- Застосування готових розцінок з довідників ПНР
- Коригування на регіональні та інші коефіцієнти
- Менш точний для нестандартних систем (IoT)

**KRTM:** Метод 1 (ресурсний) рекомендований - немає готових розцінок для ThingsBoard/IoT Gateway

### 7.3.2. Норми ЗВВ та КП для ПНР

**Згідно Настанови (Розділ IV, п. 4.3):**

| Показник | Норматив | База              |
| -------- | -------- | ----------------- |
| **ЗВВ**  | 120-140% | від ФОП інженерів |
| **КП**   | 65-75%   | від ФОП інженерів |

**Обґрунтування високих норм:**

- Висока кваліфікація інженерів (сертифікація, досвід)
- Складність робіт (програмування, налаштування, інтеграція)
- Інтелектуальна праця (діагностика, troubleshooting)
- Відповідальність за результат (працююча система, а не кількість годин)

**KRTM:**

- ЗВВ: **125%** (середнє значення діапазону)
- КП: **70%** (середнє значення діапазону)

### 7.3.3. Додаткові витрати для ПНР

**Відрядження інженерів:**

- Проїзд до віддалених котелень
- Добові витрати
- Проживання (якщо потрібне)

**KRTM:**

- 65 котелень по всьому місту Кривий Ріг
- Відрядження не потрібне (в межах міста)
- Транспортні витрати: включені в ЗВВ

**Спеціалізовані інструменти та обладнання:**

- Осцилографи, логічні аналізатори
- Генератори сигналів, калібратори
- Програматори, дебаггери

**KRTM:**

- Ноутбуки з необхідним ПЗ (Linux, Docker, MQTT clients)
- USB-UART адаптери (для Raspberry Pi console)
- Кабельні тестери, мультиметри
- Амортизація: включена в ПВ_ПНР

## 7.5. Приймання результатів ПНР

### 7.5.1. Критерії успішності ПНР

**Система вважається успішно налагодженою, якщо:**

1. **Функціональні вимоги:**
   - Всі функції відповідно до ТЗ (Tom 1) працюють коректно
   - Параметри в межах проектних значень

2. **Надійність:**
   - Безперервна робота 24-48 годин без помилок
   - Автоматичне відновлення після збоїв (reboot, network loss)

3. **Продуктивність:**
   - Затримка передачі даних < проектної (наприклад, < 5 с)
   - Пропускна здатність достатня (65 Edge × 10 датчиків × 1 хв)

4. **Безпека:**
   - Шифрування даних (MQTT over TLS 1.3+)
   - Автентифікація (X.509 сертифікати)
   - Відсутність вразливостей (security scan)

5. **Документація:**
   - Виконавча документація (as-built configs)
   - Протоколи випробувань (всі тести пройдені)
   - Інструкції для персоналу

### 7.5.2. Акт приймання ПНР

**Форма акта:** Див. Розділ VI, п. 6.5 (Акт приймання виконаних ПНР)

**Підписують:**

- Інженер-налагоджувач (виконавець)
- Представник замовника (технічний нагляд)
- Представник експлуатації (начальник котельні або диспетчер ЦДП)
- Архітектор системи (авторський нагляд, якщо залучений)

**KRTM:** Обов'язкове підписання після успішних випробувань (критерій переходу до Pilot Operation)

### 7.5.3. Гарантійні зобов'язання на ПНР

**Гарантійний термін:**

- Типово: 12-24 місяці з дати підписання акта приймання ПНР
- KRTM: **24 місяці** (відповідно Tom 1, Section 8)

**Гарантійні зобов'язання включають:**

- Усунення дефектів налагодження (безкоштовно)
- Консультації з експлуатації (телефон, email, remote access)
- Періодичні перевірки стану системи (опціонально)

**Не включають:**

- Пошкодження через неправильну експлуатацію
- Зміни конфігурації без узгодження з налагоджувачем
- Вихід з ладу обладнання (окрема гарантія виробника)

## 7.7. Особливості для різних видів систем

### 7.7.1. Системи автоматизації (SCADA, IoT)

**Характерні роботи:**

- Програмування PLC, Gateway, контролерів
- Налаштування протоколів (Modbus, OPC-UA, MQTT)
- Створення HMI/Dashboard
- Інтеграція з існуючими системами

**Трудомісткість:**

- Висока (складне програмування, налаштування)
- Залежить від кількості точок (датчиків, виконавчих механізмів)

**KRTM:** Тип системи - IoT диспетчеризація (ThingsBoard + Gateway)

### 7.7.2. Електротехнічні системи

**Характерні роботи:**

- Вимірювання опорів ізоляції
- Перевірка релейних захистів
- Налаштування трансформаторів
- Фазування

**Трудомісткість:**

- Середня
- Залежить від потужності та складності

**KRTM:** Обмежено застосовно (основні електросистеми котелень вже існують)

### 7.7.3. Системи теплопостачання

**Характерні роботи:**

- Гідравлічні випробування
- Налаштування регуляторів тиску та температури
- Балансування системи
- Перевірка приладів обліку тепла

**Трудомісткість:**

- Висока (фізичні вимірювання, налаштування)

**KRTM:** Часткове застосування (інтеграція з існуючими КВП, без зміни гідравліки)

## 7.8. Документація ПНР

### 7.8.1. Обов'язкова документація

**Технічні протоколи:**

- Протокол автономних випробувань (кожен компонент)
- Протокол комплексних випробувань (система в цілому)
- Протокол приймальних випробувань (з участю замовника)

**Виконавча документація:**

- As-built конфігурації (gateway.yaml, modbus.json тощо)
- Схеми підключення (актуальні креслення)
- Таблиці адрес та параметрів (IP адреси, Modbus slave ID, регістри)

**Інструкції:**

- Інструкція з експлуатації (для персоналу котелень)
- Інструкція з технічного обслуговування (для служби ІТ/АСУ)
- Інструкція з troubleshooting (типові несправності та усунення)
